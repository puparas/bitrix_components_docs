# Создание и редактирование форума

Форма применяется как для создания новых, так и для редактирования существующих форумов.

  

**Примечание**: правами на создание новых форумов обладают только администраторы сайта.

  

### Контекстная панель

| Кнопка | Описание |
| --- | --- |
| Список форумов | Переход на страницу со [списком](/user_help/service/forum/forum_admin.php) форумов. |
| Создать новый форум | Переход к форме создания нового форума. Кнопка отображается при редактировании существующего форума. |
| Удалить форум | Удаление редактируемого форума. |

### Вкладка Форум

Настройка общих параметров форума.

| Поле | Описание |
| --- | --- |
| Активность | Признак активности форума. Неактивные форумы доступны только тем пользователям, которые имеют права на полный доступ к ним. |
| Группа форумов | Группа, к которой относится форум. Может быть не установлена (т.е. форум без группы). |
| \*Сортировка | Относительный "вес", определяющий порядок показа форумов в рамках одной группы. Чем больше значение поля, тем ниже в списке будет расположен форум. |
| \*Имя форума | Произвольное название форума. |
| Описание | Краткое описание форума. Описание будет показано в публичной части под названием форума. |
| Сайты, на которых доступен форум / Шаблон пути к сообщению на сайте | |
| <Сайт> | Выбираются сайты, на которых будет доступен форум. |
| Шаблон пути к сообщению на сайте | Шаблон пути к сообщениям форума для данного сайта. Это поле используется при отправке почтовых сообщений с форума и при индексировании сообщений форума в модуле поиска.     Пример шаблона:    ``` /forum/index.php?   ```   Переменные **#MESSAGE\_ID#**, **#MID#**, **#TOPIC\_ID#**, **#TID#**, **#TITLE\_SEO#**, **#FORUM\_ID#**, **#FID#**, **#PARAM1#**, **#PARAM2#**, **#SOCNET\_GROUP\_ID#**, **#OWNER\_ID#** при использовании шаблона автоматически заменяются на реальные код форума, код темы, код сообщения и т.д. |

\* - Поля, обязательные для заполнения.

### Вкладка Настройки

| Поле | Описание |
| --- | --- |
| Индексация | При отмеченной опции сообщения форума будут индексироваться модулем поиска. **Примечание**: При изменении значения может потребоваться переиндексация форума |
| Модерация | Включение режима премодерирования: сообщения пользователей будут публиковаться на сайте только после проверки модератором (администратором сайта или другим пользователем с соответствующими правами). |
| Одинаковые сообщения в теме | При отмеченной опции запрещается писать два одинаковых сообщения в теме, независимо от очередности сообщений. |
| Сортировать по | Сортировка "по умолчанию" тем в рамках одного форума в публичной части сайта. Выберите вариант сортировки сообщений, а также направление сортировки. Доступна сортировка по:  * **дате последнего сообщения**; * **тема сообщения**; * **количество ответов**; * **количество просмотров**; * **дата начала темы** * **автор темы**; |
| Направление сортировки | Направление сортировки для выбранного режима сортировки. |
| Прикреплять первое сообщение темы | При отмеченной опции будет прикрепляться первое сообщение темы. |
| Позволить пользовательские подписи под сообщением | При отмеченной опции под сообщением будет автоматически отображаться подпись пользователя. |
| Позволить загрузку | Разрешить загрузку присоединенных к сообщению файлов указанного типа. Возможные варианты значений:  * **нет** - загрузка запрещена; * **изображения** - загрузка только изображений (gif, jpg, png, jpe, jpeg); * **файлов с указанными расширениями** - загрузка только файлов с расширениями из поля **Допустимые расширения загружаемых файлов**; * **любых файлов** - загрузка любых файлов. Не разрешайте без крайней необходимости загрузку выполнимых файлов и скриптов, так как это может позволить злоумышленникам полностью разрушить ваш сайт. |
| Допустимые расширения загружаемых файлов (для загрузки файлов с указанными расширениями, расширения разделять запятой) | Расширения файлов, разрешенных к загрузке на сайт (см. описание предыдущего поля). |
| Пользовательские настройки | |
| Запросить E-Mail у гостя | Определяет, будет ли запрашиваться адрес почты у незарегистрированного пользователя при попытке добавления им сообщения. |
| Использовать CAPTCHA | Определяет, использовать ли защиту от автоматического заполнения. Используется только для неавторизованных пользователей |
| Регистрация новых сообщений в модуле статистики\*\* | |
| event1 | Идентификатор типа события - **event1** |
| event2 | Идентификатор типа события - **event2** |
| event3 | Дополнительный параметр события - **event3** |
| Настройки парсера | |
| Смайлы | Разрешить использование смайликов в сообщении. |
| HTML-код | Разрешение или запрещение использования **HTML**-кода в тексте сообщения. Не рекомендуется разрешать использование произвольного **HTML**-кода на форуме без особой необходимости, так как это существенно снижает безопасность форума. При включенной опции работают **BBCode**. |
| Дополнительные настройки | Дополнительные возможности настроек форума:  * **Позволить ссылки** - Разрешить использование гиперссылок в сообщении (`<a href=...>`). * **Позволить теги B, U, I, S** - Разрешить выделение текста полужирным, подчеркиванием, курсивом или перечеркивание(`<b> <u> <i> <s>`). * **Позволить изображения** - Разрешить использование изображений в тексте сообщения. Изображения располагаются на сторонних сайтах и подключаются на форуме (`<img src=...>`). * **Позволить видео** - Разрешается использование видео в сообщениях форума. * **Позволить списки** - Разрешить создание списков (`<ul><li>`). * **Позволить таблицы** - Разрешить использование таблиц (`<table>`). * **Позволить выравнивание** - Разрешить использование выравнивания текста. * **Позволить цитирование** - Возможность цитировать сообщение другого пользователя (). * **Позволить коды** - Использование кодов в сообщении (`<code>`). * **Позволить шрифты** - Возможность изменения цвета текста и шрифт (`<font color=...>`). * **Заменять символ перевода каретки на <br>** - замена символа перевода каретки на `<br>` (Доступно только при отмеченной опции `HTML-код`). |

\*\* Секция служит для настройки идентификаторов, с использованием которых событие создания нового сообщения на форуме будет регистрироваться в модуле статистики.

### Вкладка Доступ

Предназначена для установки прав доступа к форуму для каждой из зарегистрированных на сайте групп пользователей. По умолчанию права имеют следующие значения.

| Поле | Описание |
| --- | --- |
| [Группа пользователей] | Уровень доступа к форуму для данной группы пользователей:   * **Нет доступа** - запрет на доступ к форуму; * **Чтение** - доступ на чтения сообщений форума без права создания новых тем; * **Ответ** - доступ к обсуждению существующих тем без права создания новых тем; * **Новая тема** - доступ к обсуждению существующих тем, а также возможность создания новых тем; * **Модерирование** - возможность модерировать форум, т.е. производить те действия, которые не повлекут реальной потери данных. Например, модераторы могут скрыть или показать сообщение, но не могут отредактировать или удалить его. Члены группы с данным уровнем прав имеют возможность подписаться на получение копии всех сообщений форума по почте. * **Редактирование** - доступ к управлению чужими сообщениями на форуме; * **Полный доступ** - полный доступ к форуму, включая доступ к настройкам форума и прав доступа.   **Примечание**: Младшие права включаются в старшие (т.е. если группа имеет право на модерирование форума, то она тем более имеет право читать сообщения, отвечать на сообщения, создавать новые темы, но не имеет права на редактирование сообщений). При определении прав пользователя, который принадлежит к нескольким группам, берется максимальное из его прав. |
