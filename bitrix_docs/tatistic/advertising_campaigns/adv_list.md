# Рекламные кампании. Список кампаний

На странице **Рекламные кампании** (*Аналитика > Рекламные кампании > Список кампаний*) представлены обзорные данные по рекламным кампаниям.

  

### Фильтр

| Поле | Описание |
| --- | --- |
| **Найти** | Поле, позволяющее найти рекламные кампании по их основным параметрам: **идентификатору**, **referer1** или **referer2**.   Это поле присутствует, даже если фильтр свернут. |
| ID\* | ID рекламных кампаний, которые требуется отобразить. |
| referer1 / referer2\* | Идентификаторы рекламных кампаний, соответственно **referer1** и **referer2**. |
| Длительность (дней) | Интервал для фильтрации по длительности рекламной кампании. Длительность - время, прошедшее с момента первого захода по рекламной кампании до последнего прямого захода либо возврата. |
| Сессий (всего) | Интервал для фильтрации по суммарному значению сессий (на прямом заходе либо возврате). |
| Посетителей (всего) | Интервал для фильтрации по суммарному значению посетителей (на прямом заходе либо возврате). |
| Период (DD.MM.YYYY) | Установка данного поля фильтра выводит дополнительный столбец в отчете по рекламным кампаниям, в котором содержатся данные за установленный период. |
| Внимательность | Интервал для фильтрации по значению **внимательности** (среднее значение количества хитов за одну сессию). |
| Посетителей за день | Интервал для фильтрации по количеству посетителей за день (среднее значение количества посетителей за один день). |
| Затраты | Интервал для фильтрации по полю рекламной кампании **затраты**. В данном поле может храниться денежная сумма, затраченная на рекламную кампанию. |
| Доходы | Интервал для фильтрации по полю **доходы**. Данное поле накапливает все денежные суммы по каждому событию, произошедшему в рамках рекламной кампании. Значение поля может быть изменено вручную. |
| Прибыль | Интервал для фильтрации по полю **прибыль** (разница между доходами и затратами на рекламную кампанию). |
| Рентабельность (ROI) | Интервал для рентабельности рекламной кампании, рассчитывается по формуле: **`(прибыль/затраты)*100%`**. |

\* - для данных полей вы можете воспользоваться [специальными логическими выражениями](https://dev.1c-bitrix.ru/api_help/main/general/filter.php).

### Контекстная панель

| Кнопка | Описание |
| --- | --- |
| Добавить | Нажатие этой кнопки открывает форму создания новой рекламной кампании. |
| Группировка | Позволяет сгруппировать данные таблицы по одному из двух идентификаторов рекламной кампании - **referer1** либо **referer2**, или **отменить группировку**. |
| Настроить | Переход к диалогу настройки внешнего вида отчетной формы. |
| Excel | Экспорт данных из отображаемой таблицы в **MS Excel**. |

### Таблица рекламных кампаний

| Поле | Описание |
| --- | --- |
| Колонка флажков | Поле предназначено для выбора кампаний, к которым предполагается применить какое-либо действие из панели действий, расположенной ниже таблицы кампаний. |
| Меню действий | Нажатие на кнопку действий в строке конкретной кампании отображает меню, содержащее команды:  * **Анализ рекламной кампании** - открывает [форму](/user_help/statistic/advertising_campaigns/adv_detail.php), отображающую детальную информацию по динамике рекламной кампании: сводную статистику, графики, финансовые показатели, события и проч.; * **Изменить** - открывает форму редактирования рекламной кампании; * **Сбросить** - сбрасывает счетчики рекламной кампании: кол-во посетителей, внимательность, финансовые индикаторы; * **Удалить** - удаляет рекламную кампанию;  Следующие пункты меню присутствуют, если по данной рекламной кампании произошло по крайней мере одно событие (например, хит):  * **Разделы** - открывает форму с отчетом и диаграммой посещаемости разделов и страниц посетителями, пришедшими по этой рекламной кампании; * **Пути** - открывает форму с отчетом и диаграммой путей, по которым прошли посетителями; * **Динамика** - отображает показатели динамики рекламной кампании по дням (сессии, посетители и т.п.); * **Графики** - открывает [форму с графиками](/user_help/statistic/advertising_campaigns/adv_graph_list.php) по рекламной кампании (посещения и события). |
| ID | ID рекламной кампании. |
| referer1 | Идентификатор рекламной кампании – **referer1**. |
| referer2 | Идентификатор рекламной кампании – **referer2**. |
| Начало | Время первого захода по рекламной кампании. |
| Окончание | Время последнего захода либо возврата по рекламной кампании. |
| Приоритет | Приоритет рекламной кампании. Это значение вводится при создании (или редактировании) кампании. |
| Зашли (сессий) сегодня | Кол-во сессий на прямом заходе (переходов) и возврате\* за текущий день. |
| Зашли (сессий) всего | Суммарные значения количеств сессий на прямом заходе и на возврате. |
| Всего событий сегодня | Количество событий на прямом заходе и возврате за текущий день. |
| Всего событий | Количество событий за период (если он установлен в фильтре). |

\* - **Возврат по рекламной кампании** подразумевает ситуацию, когда посетитель делал непосредственный (прямой) заход по рекламной кампании и спустя некоторое время вернулся на сайт, при условии, что в этот промежуток времени он больше не делал прямых заходов по другим рекламным кампаниям.

### Дополнительная информация: рекламные кампании

Под термином **рекламная кампания** подразумевается набор условий, выделяющих часть потока посетителей, приходящих на Ваш сайт.  Все условия равносильны и могут быть следующих типов:

* Наличие в ссылке, ведущей на Ваш сайт, одного или двух параметров – **referer1**, **referer2**.   
  Пример подобной ссылки: **`http://www.1c-bitrix.ru/?referer1=vesti&referer2=news`**.
* Поисковые системы, с которых могут заходить посетители.
* Маски любых ссылающихся страниц.
* Маски страниц Вашего сайта, на которые приходят посетители.

При определении рекламной кампании используется следующий алгоритм.

1. Определяются те рекламные кампании, условия которых удовлетворяют параметрам посетителя.
2. Среди выбранных рекламных кампаний берется кампания с наивысшим приоритетом (задается в настройках рекламной кампании). Если приоритеты одинаковы, то берется самая молодая рекламная кампания (с наибольшим ID).

Если посетитель пришел на сайт с параметрами **referer1** и/или **referer2**, и с такими идентификаторами не было найдено ни одной рекламной кампании, то автоматически создаётся новая рекламная кампания с указанными **referer1**, **referer2** в качестве идентификаторов.
