# Создание и редактирование баннера

Форма предназначена для создания новых и изменения параметров существующих баннеров. Для удобства работы форма разделена на несколько закладок. Каждая закладка содержит логически сгруппированный набор полей для управления параметрами баннера.

### Баннер

Закладка cлужит для задания основных параметров баннера.

| Поле | Описание |
| --- | --- |
| Статус баннера | Индикатор, определяющий статус показа баннера: - баннер не показывается;  - баннер показывается. Поле отображается при редактировании существующего баннера. |
| Создан | Дата и время создания баннера, а также имя пользователя, создавшего баннер.     Поле отображается при редактировании существующего баннера. |
| Изменен | Дата и время изменения баннера, а также имя пользователя, выполнившего изменения.     Поле отображается при редактировании существующего баннера. |
| Контракт | Указывается контракт, к которому приписывается баннер. Перечень доступных контрактов представлен в виде поля со списком. |
| Активен | При отмеченной опции баннер будет активным. Для показа в публичном разделе доступны только активные баннеры. |
| Интервал показа (DD.MM.YYYY) | Задается период времени, в течение которого баннер может показываться на сайте. По истечении этого периода, баннер становится не активным. |
| Название | Указывается произвольное имя баннера. |
| Группа баннеров | Указывается произвольная группа баннера (в выпадающем списке вы можете выбрать одну из ранее созданных групп). |
| Тип баннера | Указываетя тип баннера. |
| Вес (приоритет) | Задается относительный "вес", влияющий на вероятность выбора баннера для показа в публичном разделе сайта. |
| Что показывать | |
| Тип шаблона баннера | Способ формирования внешнего вида баннера. В зависимости от выбранного значения (**Изображение**, **Flash** (больше не поддерживается    Поддержка Flash-технологий официально прекращена 31.12.2020. ), **HTML-код**, **Баннер для шаблона Bootstrap**, **Баннер для шаблона Niva Slader**, **Баннер для шаблона Jssor Slader**, **Баннер для шаблона Parralax**, **Баннер для шаблона Bootstrap 4**.) количество полей в секции **Что показывать** будет различаться. В шаблона для слайдеров и в шаблоне для Bootstrap создаётся неограниченное число слайдеров. |
| Расширенный режим | Изменяет возможности настроек выбранного типа баннера. Неактуален для **Изображения**, **Flash** и **HTML-кода**. Расширенный режим показывает html-редактор, в котором можно сделать какие угодно варианты отображения слайда, а не по заданным в системе шаблонам. |
| Изображение | |
| Файл | Поле для загрузки графического файла с локального компьютера. |
| URL для перехода при клике на изображении | Адрес страницы, которая будет открыта по клику на баннере. В параметрах URL'а можно использовать шаблон **#EVENT\_GID#**, заменяемый на строку, идентифицирующую посетителя (используется модулем **Веб-аналитика**). |
| Где развернуть URL (атрибут target) | Параметр, определяющий, в каком окне будет показана страница, открытая по клику на баннер:  * **\_self** – открыть в текущем окне (по умолчанию); * **\_blank** – открыть в новом окне; * **\_parent** – открыть в своем фрейме; * **\_top** - во всем текущем окне браузера. |
| Текст всплывающей подсказки | Текст подсказки, которая будет появляться при наведении указателя мыши на баннер. |
| Показать/скрыть дополнительный HTML-код баннера | Ссылка позволяет показать либо скрыть поле для дополнительного HTML-кода баннера.     **Код баннера** - произвольный текст или **HTML**-код, который будет отображаться в качестве баннера.     В коде баннера можно использовать макросы **#RANDOM1#**, **#RANDOM2#**, **#RANDOM3#**, **#RANDOM4#**, **#RANDOM5#**, которые будут заменены случайными числами в момент показа в публичной части сайта данного кода. Также можно использовать следующие шаблоны:  * **#BANNER\_NAME#** - имя баннера; * **#BANNER\_ID#** - ID баннера; * **#CONTRACT\_ID#** - ID контракта; * **#TYPE\_SID#** - тип баннера. |

### Ограничения

Закладка служит для задания ограничений на показ баннера.

| Поле | Описание |
| --- | --- |
| Фиксировать число показов баннера | При отмеченной опции число показов баннера будет фиксироваться (ограничиваться). |
| Показывать баннеры с равномерной скоростью | При отмеченной опции баннер будет показываться с равномерной скоростью.   Для этого необходимо настроить следующие параметры:  * указать **Интервал показа (DD.MM.YYYY)**; * отметить опцию **Фиксировать число показов баннера**; * задать **Максимальное число показов баннера**. |
| Максимальное число посетителей, для которых будет показан баннер | Указывается максимальное число посетителей, для которых будет показан баннер.  Это поле будет активно только при отмеченной опции **Фиксировать число показов баннера**. |
| Максимальное число показов данного баннера одному посетителю | Указывается максимальное число показов данного баннера одному посетителю. Считается, что оптимальным количество показов баннера одному посетителю равно 4.   Это поле будет активно только при отмеченной опции **Фиксировать число показов баннера**. |
| Максимально число показов баннера | Указывается максимально число показов баннера, при достижении которого баннер становится неактивным.     При редактировании баннера рядом с данным полем отображается число выполненных показов баннера. Чтобы обнулить показатели счетчика, установите флаг в поле **сбросить** и нажмите кнопку **Применить**.  Это поле будет активно только при отмеченной опции **Фиксировать число показов баннера**. |
| Фиксировать клики на данный баннер    Доступно до версии 21.100.0 | При отмеченной опции клики по данному баннеру будут регистрироваться в модуле **Веб-аналитика**. |
| Максимальное число кликов на баннер    Доступно до версии 21.100.0 | Указывается максимальное число кликов на баннер, при достижении которого баннер становится неактивным.     При редактировании баннера рядом с данным полем отображается число выполненных кликов по баннеру. Для того чтобы обнулить показатели счетчика, установите флаг в поле **сбросить** и нажмите кнопку **Применить**. |
| Отклик баннера (CTR) (%)    Доступно до версии 21.100.0 | В поле отображается значение показателя отклика данного баннера в процентах. |

### Таргетинг

Закладка служит для настройки таргетинга рекламы. Диапазон доступных значений полей зависит от настроек контракта, к которому относиться данный баннер.

| Поле | Описание |
| --- | --- |
| Сайты | Указываются сайты, на которых будет показываться баннер. |
| Разделы и страницы, на которых баннер обязательно будет показан | Указываются [адреса разделов](#slash) или страниц сайта, на которых баннер будет обязательно показан. Например: **/ru/catalog/** или **/ru/catalog/index.php**.   Или, чтобы показать баннер **только** на главной странице сайта, нужно задать в первом поле полный адрес страницы (не раздела, а именно включая файл страницы): **www.mysite.ru/index.php**. |
| Разделы и страницы, на которых баннер не будет показан | Указываются [адреса разделов](#slash) или страниц сайта, на которых не будет показан данный баннер.     Адреса страниц могут быть заданы с использованием параметров. Каждый адрес раздела или страницы должен начинаться с новой строки. |
| Группы пользователей | Указываются группы пользователей, для которых будет/не будет показываться данный баннер.     Для этого выберите необходимые группы пользователей и установите переключатель в одно из значений: **Показывать баннер только для выбранных групп** или **Не показывать баннер для выбранных групп**. |
| Ключевые слова | Задаются ключевые слова для баннера, влияющие на возможность показа баннера на страницах сайта:  * если для страницы сайта заданы **желательные ключевые слова**, то баннер будет показан на странице в том случае, если хотя бы одно ключевое слово баннера присутствует в наборе желательных ключевых слов страницы; * если для страницы сайта заданы **обязательные ключевые слова**, то баннер будет показан на странице в том случае, если набор ключевых слов баннера совпадает с набором обязательных ключевых слов страницы.  Более подробная информация приводится в руководстве [Управление показом баннеров с помощью ключевых слов](http://www.1c-bitrix.ru/download/manuals/ru/banner_show_tutorial.doc). |
| Параметры геотаргетинга | Указываются страны, регионы или города, посетителям из которых будет показываться данный баннер.     Чтобы добавить, например страну, в список, установите опцию **Страны** и нажмите кнопку **Добавить...** и в открывшейся форме укажите страну, которую требуется добавить. Для удаления выберите пункт(ы) в списке и нажмите кнопку **Удалить отмеченные**. |
| Показать только для посетителей, приходящих по следующим рекламным кампаниям | **Поле доступно только администраторам модуля**.     Указываются рекламные компании, для посетителей которых (пришедших по определённой рекламной компании) будет показан баннер. Регистрация рекламных компаний выполняется с помощью модуля **Веб-аналитика**. |
| Показывать посетителям | Указывается каким посетителям показывать баннер:  * **всем** - показывать баннер всем посетителям сайта; * **только новым** - показывать баннер пользователям, впервые пришедшим на сайт; * **только вернувшимся** - показывать баннер пользователям, вернувшимся на сайт. |
| Время и дни недели для показа | Задаются часы и дни недели, по которым баннер доступен для показа. На диапазон доступных значений влияют ограничения, заданные для [контракта](/user_help/marketing/advertising/adv_contract_edit.php), к которому относится баннер. |

**Примечание:** наличие слэша в конце адреса влияет на выбор каталогов системой, так как включаются все адреса, начинающиеся с указанной строки.

Пример:

* Если ввести: **www.mysite.ru/dir/**, то под условие будут подходить все страницы из разделов *www.mysite.ru/dir/*, *www.mysite.ru/dir/other/* и т.п.
* Если ввести **www.mysite.ru/dir**, то под условие будут подходить все страницы предыдущего примера, плюс к ним добавятся разделы вида *www.mysite.ru/dir\_another*, *www.mysite.ru/directory*, и т.п., то есть, отношения к каталогу **dir** не имеющие.

### Статистика

**Внимание!** С версии **21.100.0** модуля **Реклама, баннеры** убрана возможность фиксировать статистику по кликам на баннеры, т.е. вкладка **Статистика** в форме настроек баннера более НЕ доступна. Возможность считать клики на баннер оставлена в [API](https://dev.1c-bitrix.ru/api_help/advertising/classes/cadvbanner/click.php) (требуется написание своего скрипта для подсчета ссылок).

Закладка служит для управления параметрами регистрации кликов на баннер в модуле **Веб-аналитика**. Управление параметрами доступно только администраторам сайта.

| Поле | Описание |
| --- | --- |
| Фиксировать в статистике | При отмеченной опции будет происходить регистрация событий клика по баннеру в модуле **Веб-аналитика**. |
| Event1 | Идентификатор типа события - **event1**. |
| Event2 | Идентификатор типа события - **event2**. |
| Event3 | Дополнительный параметр события.     Для создания идентификаторов типа события **event1**, **event2**, **event3** можно использовать следующие макросы:  * **#BANNER\_NAME#** - имя баннера; * **#BANNER\_ID#** - ID баннера; * **#CONTRACT\_ID#** - ID контракта; * **#TYPE\_SID#** - тип баннера. |
