# Создание и редактирование пользовательского поля

Форма предназначена для управления параметрами пользовательского поля.

### Контекстная панель

| Кнопка | Описание |
| --- | --- |
| Список | Переход к [списку пользовательских полей](/user_help/settings/settings/userfield_admin.php). |
| Добавить | Переход к добавлению нового пользовательского поля. |
| Удалить | Удаление редактируемого пользовательского поля.     Кнопка отображается при редактировании существующего пользовательского поля. |

### Форма редактирования

| Поле | Описание |
| --- | --- |
| ID | Уникальный идентификатор пользовательского поля. Присваивается после сохранения. |
| \*Тип данных (можно задать только для нового поля) | Выбирается один из возможных типов данных: - [**Привязка к элементам highload-блоков**](#highload) (для коробочной версии Битрикс24) — связывает данные с записями из highload-блоков (таблиц для хранения структурированной информации). - [**Документ из библиотеки документов**](#webdav) (для коробочной версии Битрикс24) — выбор файла из библиотеки документов для работы с внутренними файлами. - [**Число**](#number) — хранит дробные и целые числовые значения, например, для рейтингов или метрик. - [**Да/Нет**](#yes) — булево поле (галочка/флажок) для простых переключателей, например, «Активен». - [**Видео**](#video) — вставка видео через URL. Автоматически отображает превью. - [**Деньги**](#currency) — используется для финансовых данных с поддержкой валют, например, сумма заказа. - [**Шаблон**](#template) — позволяет хранить и использовать предопределенные шаблоны. - [**Список**](#list) — выбор из заданных вариантов: выпадающий список, чекбоксы, радиокнопки. - [**Строка**](#line) — однострочный текст, например, название. - [**Дата со временем** или **Дата**](#time) — выбор даты и/или времени для событий, дедлайнов. - **Привязка к сотруднику** (для коробочной версии Битрикс24) — выбор пользователя из списка сотрудников, например, ответственный за задачу. - [**Привязка к разделам или элементам инф. блоков**](#section) — связь с разделами или элементами инфоблоков, например, каталог товаров. - [**Привязка к элементам CRM**](#element_CRM) (для коробочной версии Битрикс24) — связь с объектами CRM: сделки, контакты, компании. - [**Привязка к справочникам CRM**](#reference) (для коробочной версии Битрикс24) — выбор значений из CRM-справочников. - [**Файл**](#file) — загрузка файлов (документы, изображения) напрямую в поле. - [**Целое число**](#full_number) — числовое поле без дробной части, например, количество товаров. - [**Опрос**](#poll) — используется для создания и хранения данных опросов. - [**Ссылка**](#link) — используется для ввода URL-адреса. Подходит для хранения ссылок на внешние ресурсы. - [**Адрес**](#adress) — используется для ввода адреса. - [**Бронирование ресурсов**](#reservation) — предназначен для управления и учета бронирования различных ресурсов, таких как переговорные комнаты, оборудование или другие объекты, доступные для резервирования в компании. - **Версия файла (Диск)** (для коробочной версии Битрикс24) — используется для управления версиями документов и файлов, хранящихся на Битрикс24.Диск. - **Письмо** — используется для работы с электронными письмами. - **Содержимое ссылки** — поле для работы с предпросмотром ссылок. Автоматически извлекает мета-данные (заголовок, описание, изображение) и отображает превью ссылок. - **Файл (Диск)** (для коробочной версии Битрикс24) — загрузка файлов в хранилище Битрикс24.Диск.   **Примечание**: в зависимости от выбранного типа секция **Дополнительные настройки поля** будет иметь различный вид. |
| \*Объект | Код объекта, к которому относится поле. Можно задать    Детальнее о Пользовательских полях, список объектов  [Подробнее ...](https://dev.1c-bitrix.ru/learning/course/index.php?COURSE_ID=43&CHAPTER_ID=04804) только для **нового поля**. |
| \*Код поля (можно задать только для нового поля) | Код самого поля. Можно задать только для **нового поля**. |
| XML\_ID | Указывается идентификатор поля для работы с внешними источниками. |
| Сортировка | Индекс сортировки, определяющий положение поля в общем списке. |
| Множественное (можно задать только для нового поля) | При отмеченной опции значение поля будет множественным. Можно задать только для **нового поля**. |
| Обязательное | При отмеченной опции поле будет обязательным для заполнения. |
| Показывать в фильтре списка | Указывается способ отображения поля в фильтре:  * **не показывать** - поле не будет показано в фильтре; * **точное совпадение** - поле будет показано в фильтре. Для нахождения элемента требуется точное совпадение условию поиска; * **поиск по маске** - поле будет показано в фильтре. Для поиска элемента в списке по данному полю можно использовать маску; * **поиск по подстроке** - поле будет показано в фильтре. Для нахождения элемента выполнятся поиск по подстроке. |
| Не показывать в списке | При отмеченной опции поле не будет отображаться в списке элементов в месте применения. |
| Не разрешать редактирование пользователем | При отмеченной опции значение поля нельзя редактировать, только вычислять и устанавливать через API. |
| Значения поля участвуют в поиске | При отмеченной опции значения поля будут участвовать в поиске в публичной части после переиндексации сайта. |

| Высота списка | Указывается высота списка. |
