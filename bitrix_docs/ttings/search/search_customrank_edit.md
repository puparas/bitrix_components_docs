# Создание и редактирование правила сортировки результатов поиска

Форма предназначена для создания нового или изменения существующего правила сортировки результатов поиска.

**Примечание**: применение правила предполагает изучение системой поиска строки запроса; нахождение документов, соответствующих запросу; поиск правил, соответствующих найденным документам; в случае нахождения подходящих правил - ранжирование результатов поиска в соответствии с весами сортировки, указанных в правилах.

Например, пользователь желает найти документы, соответствующие запросу "*телефоны nokia*", и существует правило:

**"модуль *Информационные блоки* :: тип блока *Каталог* :: блок *Телефоны Nokia* :: сорт. *100*"**

После поиска система предпримет попытку найти подходящее запросу правило и обнаружит, что некоторые результаты поиска включают в себя элементы информационного блока **Телефоны Nokia**. Таким образом, к результатам поиска будет применено вышеуказанное правило, что повлечет за собой назначение веса сортировки 100 результатам из элементов информационного блока **Телефоны Nokia**.

### Закладка Правило

| Поле | Описание |
| --- | --- |
| Сайт | Указывается сайт, в пределах которого должно действовать данное правило. Правило не будет применяться к результатам поиска на других сайтах. |
| Модуль | Указывается модуль, к элементам которого применимо данное правило. Система позволяет определять правила, которые будут применены:  * для **Cтатических файлов** - область действия правила ограничивается обязательно указываемым в поле **Параметр 3** полным путём к требуемому файлу; * для модуля **Информационные блоки** - область действия правила может быть ограничена: определённым типом инфоблоков, определённым информационным блоком, а также элементом блока. Можно создать правило без ограничений по типам или элементам информационных блоков. В таком случае правило сортировки будет распространятся на все типы и элементы всех информационных блоков. * для модуля **Блоги** - область действия правила может быть ограничена: определённым блогом, комментарием блога, сообщением блога или пользователем блога. * для модуля **Форум** - область действия правила может быть ограничена: форумом, определённой темой форума, а также сообщением. Правило сортировки будет распространятся на все сообщения всех форумов, если не указаны ни один из трех параметров правила (форум, тема, сообщение). * для модуля **Социальная сеть** - область действия правила может быть ограничена: группой соцсети, темой группы соцсети, типом инфоблока и инфоблоком, для которых определен функционал соцсети (форумы, блоги, галереи групп и пользователей). |
| Параметр 1 | Первый параметр правила. В зависимости от выбранного модуля, параметр может определять:  * для **Cтатических файлов** - не используется; * для модуля **Информационные блоки** - тип информационного блока; * для модуля **Блоги** - не используется; * для модуля **Форум** - форум; * для модуля **Социальная сеть** - не используется. |
| Параметр 2 | Второй параметр правила. Может определять:  * для **Статических файлов** - не используется; * для модуля **Информационные блоки** - информационный блок; * для модуля **Блоги** - не используется; * для модуля **Форум** - тему форума; * для модуля **Социальная сеть** - не используется. |
| Параметр 3 | Третий параметр правила. Может определять:  * для **Статических файлов** - полный путь к файлу; * для модуля **Информационные блоки** - элемент информационного блока; * для модуля **Блоги** - блог, комментарий блога, сообщение блога, пользователь блога; * для модуля **Форум** - текст в сообщении темы форума; * для модуля **Социальная сеть** - группу соцсети, тему группы соцсети, тип инфоблока и инфоблок, для которых определен функционал соцсети (форумы, блоги, галереи групп и пользователей). |
| Вес | Указывается вес (ранг) правила. Документы, соответствующие критериям правила, будут расположены в списке результатов поиска с соответствии с рангом сортировки. Чем больше значение веса, тем выше в списке будет отображен документ. |
