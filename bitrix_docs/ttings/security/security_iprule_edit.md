# Создание и редактирование правила стоп-листа

На данной странице можно создать новое или изменить параметры существующего правила блокировки доступа.

  

### Контекстная панель

| Кнопка | Описание |
| --- | --- |
| Список | Переход к [списку правил блокировки](/user_help/settings/security/security_iprule_list.php). |
| Добавить | Переход на страницу с формой создания нового правила блокировки. Кнопка отображается при редактировании существующего правила. |
| Удалить | Удаление редактируемого правила блокировки. |

### Форма редактирования

| Поле | Описание |
| --- | --- |
| Активность | При отмеченной опции запись будет активной. |
| Действует для страниц административного раздела | При отмеченной опции правило будет действовать для страниц административного раздела. |
| Действует для страниц публичной части сайта | Указывается сайт, для страниц публичной части которого будет действовать правило блокировки. |
| Сортировка | Порядок сортировки в списке правил. Чем меньше порядок сортировки, тем выше в списке будет показано правило. |
| Название | Название правила. |
| Начало активности | Дата и время начала активности правила. |
| Окончание активности | Дата и время окончания активности правила. |
| IP-адреса и диапазоны адресов, которые будут заблокированы | IP-адреса и диапазоны адресов, которые будут заблокированы. Например: **192.168.0.7** или **192.168.0.1-192.168.0.100**. Каждый IP адрес добавляется с помощью нового поля, которое добавляется с помощью кнопки **Добавить**. |
| Исключения IP из заблокированных | IP-адреса и диапазоны адресов, доступ с которых будет разрешен. |
| Маски путей, доступ к которым будет заблокирован | Маски путей разделов и страниц, доступ к которым будет заблокирован. Например: **/\*** или **/bitrix/admin/\***. |
| Исключения масок путей | Маски путей разделов и страниц, доступ к которым будет разрешен. |
